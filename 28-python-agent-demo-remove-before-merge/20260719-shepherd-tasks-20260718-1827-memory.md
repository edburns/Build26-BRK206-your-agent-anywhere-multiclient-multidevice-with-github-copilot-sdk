### Bugs Diagnosed & Fixed

1. **Idle-kill timeout bug** — The Copilot CLI runtime terminates sessions ~60s after an `assistant.idle` event fires. This happened when Phase 2 agents backgrounded their polling commands (exceeding `initial_wait: 120s`). The agent would have no pending tool calls, go idle, and get killed before the Copilot code review arrived (~180s).
   - **Fix:** Added `⚠️ CRITICAL: Never go idle while waiting` directive to all 3 skills with 4 rules: use `initial_wait ≥ 600`, never background poll commands, always have a tool call pending, emit progress messages.

2. **Missing Copilot code review request** — Marking a PR as "Ready for review" does NOT automatically trigger Copilot code review. The Phase 2 skill was waiting for a review that was never requested.
   - **Fix:** Added explicit `gh pr edit $PR_NUMBER --add-reviewer "copilot-pull-request-reviewer"` to Step 1 of `shepherd-task-from-ready-to-merged-to-base`.

3. **PR base branch merge-to-wrong-target (issue #32)** — Phase 1 agent committed directly to `edburns/28-python-agent-demo` (the BASE_BRANCH) instead of creating a child branch. This made head==base when trying to retarget. Phase 2 then rationalized the error and merged to `main`.
   - **Root cause:** The CCA didn't create a feature branch off BASE_BRANCH.
   - **Script gap:** `shepherd-task.ps1` line 153 doesn't check exit code of `gh pr edit --base`. (Not yet fixed per user decision.)

4. **OTEL file exporter not flushing (copilot-agent-runtime#13047)** — OTEL spans collected in memory but never flushed to disk when `copilot` is invoked via piped stdin. Works with `-p` flag. Bug filed.
